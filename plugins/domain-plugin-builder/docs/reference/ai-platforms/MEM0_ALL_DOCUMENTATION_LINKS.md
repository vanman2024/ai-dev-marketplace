#### Installation

**NPM Package**:
```bash
# Run with npx
```

**Python Package**:
```bash
# Clone and setup
git clone https://github.com/mem0ai/mem0-mcp
cd mem0-mcp
uv venv
source .venv/bin/activate
uv pip install -e .
```

#### Configuration

**VS Code Settings** (`settings.json`):
```json
{
  "mcp": {
    "inputs": [
      {
        "type": "promptString",
        "id": "apiKey",
        "description": "Mem0 API Key",
        "password": true
      }
    ],
    "servers": {
      "mem0-memory": {
        "command": "npx",
        "args": ["-y", "@mem0/mcp"],
        "env": {
          "MEM0_API_KEY": "${input:apiKey}"
        }
      }
    }
  }
}
```

**Cursor Configuration**:
```json
{
  "mcpServers": {
    "mem0-mcp": {
      "command": "npx",
      "args": ["-y", "@mem0/mcp"],
      "env": {
        "MEM0_API_KEY": "YOUR-API-KEY-HERE"
      }
    }
  }
}
```

## Architecture Patterns

### Memory Types
1. **User Memory**: Persistent across all conversations for a user
2. **Agent Memory**: Specific to an agent's interactions
3. **Session/Run Memory**: Temporary memory for a conversation session

### Memory Operations
1. **Add**: Store new memories from conversations
2. **Search**: Semantic search for relevant memories
3. **Update**: Modify existing memories
4. **Delete**: Remove specific or all memories
5. **Get**: Retrieve memories by ID or filters

### Advanced Features
- **Graph Memory**: Track relationships between memories
- **Rerankers**: Improve retrieval relevance
- **Filters**: Complex logical queries (AND, OR, IN, NOT)
- **Metadata**: Custom categorization and tagging
- **Timestamps**: Automatic or manual timestamp tracking
- **Expiration**: Time-based memory removal

## Notes for AI Agents

### When to Use Mem0
✅ Building conversational AI with context retention
✅ Personalized user experiences
✅ Multi-agent systems with shared knowledge
✅ Customer support bots that remember users
✅ AI tutors that adapt to learning styles

### Platform vs Open Source Decision
- **Use Platform if**: You want managed infrastructure, enterprise features, quick deployment
- **Use Open Source if**: You need full control, custom components, self-hosting requirements
