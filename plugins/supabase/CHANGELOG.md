# Changelog

All notable changes to the Supabase plugin will be documented in this file.

The format is based on [Keep a Changelog](https://keepachangelog.com/en/1.0.0/)
and this project adheres to [Semantic Versioning](https://semver.org/spec/v2.0.0.html).

## [1.0.0] - 2025-10-26

### Added
- Initial release of Supabase plugin for Claude Code
- 16 slash commands for Supabase integration
- 14 specialized agents for multi-layer orchestration
- 7 skills with scripts, templates, and examples
- Complete AI application setup with pgvector
- Multi-agent parallel execution patterns
- Row Level Security (RLS) templates and testing
- Authentication provider configuration
- Schema generation for chat, RAG, and multi-tenant apps
- End-to-end testing workflows
- Comprehensive validation and security auditing
- Production deployment readiness checks

### Commands
- `/supabase:init` - Initialize Supabase in project
- `/supabase:init-ai-app` - Complete AI app setup
- `/supabase:setup-ai` - AI features with parallel agents
- `/supabase:setup-pgvector` - pgvector configuration
- `/supabase:add-auth` - Authentication with parallel validation
- `/supabase:add-storage` - Storage buckets
- `/supabase:add-realtime` - Realtime subscriptions
- `/supabase:add-rls` - Row Level Security
- `/supabase:add-ui-components` - Supabase UI integration
- `/supabase:create-schema` - Database schema generation
- `/supabase:deploy-migration` - Migration deployment
- `/supabase:generate-types` - TypeScript type generation
- `/supabase:validate-setup` - Configuration validation with parallel checks
- `/supabase:validate-schema` - Schema validation
- `/supabase:test-e2e` - End-to-end tests with parallel execution
- `/supabase:test-rls` - RLS policy testing
