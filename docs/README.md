## üìù Documentation Standards

### File Naming

- Use UPPERCASE-WITH-HYPHENS.md for main documents
- Use lowercase-with-hyphens.md for subdocuments

### Structure

Each document should include:

1. Title and description
2. Table of contents (for longer docs)
3. Clear sections with headers
4. Code examples where applicable
5. Links to related documentation

### Updates

- Date all major updates
- Keep changelog at bottom of document
- Link related documents bidirectionally
